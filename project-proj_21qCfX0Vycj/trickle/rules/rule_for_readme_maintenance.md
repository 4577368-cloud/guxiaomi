# 文档维护规则

## 何时更新README.md

当以下情况发生时需要更新README.md文档：

- 添加新功能或组件时
- 修改现有功能的工作方式时  
- 更新技术栈或依赖项时
- 修改文件结构时
- 添加新的API集成时
- 更改数据存储方式时
- 修复重要bug时
- 更新使用方法或操作流程时

## 更新内容包括

- 功能描述的准确性
- 技术架构的最新状态
- 文件结构的当前组织方式
- 使用方法的详细步骤
- 注意事项和已知限制
- 版本更新日志

## 何时更新DEVELOPMENT_GUIDE.md

当以下情况发生时需要更新开发文档：

- 添加新的核心功能模块
- 修改重要算法或计算逻辑
- 更新API集成方式
- 修改数据模型结构
- 添加新的工具函数
- 优化性能或架构
- 修复影响开发的重要问题

## 开发文档更新内容

- 技术架构详解
- 核心算法说明
- API使用方法
- 数据模型定义
- 性能优化策略
- 开发注意事项
- 常见问题解答
