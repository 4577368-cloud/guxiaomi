# 功能验证说明

## 前提

1. **启动服务**（在 `guxiaomi` 目录）：
   ```bash
   python3 run_web.py
   ```
2. 用浏览器打开：**http://localhost:8888/index.html**（不要用 `file://` 打开本地 HTML，否则部分功能可能异常）。

---

## 一、每只股票旁的「分析」「新闻」按钮

1. 打开 **http://localhost:8888/index.html**
2. 若组合里还没有股票：点「新增股票」，添加一只（如代码 `03690`，市场选港股）。
3. 在**每只股票卡片的右上角**（股票代码同一行）应看到一排按钮，从左到右依次为：
   - **分析**（蓝/青）
   - **新闻**（粉红）
   - 折叠/展开（三角）
   - 刷新（绿色）
   - 删除（红色）
4. **验证「分析」**：点击该只股票的「分析」→ 应跳转到分析页，且**股票代码、市场、股票名称（若有）已自动填好**。
5. **验证「新闻」**：点击「新闻」→ 应跳转到新闻页，且地址栏带有 `?code=xxx&market=xxx`（名称若有则还有 `&name=xxx`）。当前新闻页内按股票搜索的逻辑是占位，后续可再接。

**若看不到「分析」「新闻」**：确认访问的是 http://localhost:8888/index.html，并强制刷新（Ctrl+F5 或 Cmd+Shift+R）清缓存后再看。

---

## 二、分析页自动带出股票参数

不通过组合页，直接打开下面链接：

- **http://localhost:8888/analysis.html?code=03690&market=港股&name=美团**

打开后检查：

- 「股票代码」应为 `03690`
- 「市场」应为 `港股`
- 「股票名称」应为 `美团`

说明分析页已正确读取 URL 参数并预填表单。

---

## 三、投资组合总览与股票卡片更紧凑

1. 在 **http://localhost:8888/index.html** 看「投资组合总览」：
   - 资金池、总投入、剩余资金、当前市值、总盈亏应在一行或两行内展示，高度比之前更矮。
2. 点开任意一只股票（若已折叠则先展开）：
   - 购入渠道、市场、当前价格在同一行，且**不再重复显示「股票代码」**。
   - 行情数据（开盘/最高/最低/成交量/涨跌等）在一行或两行内联显示。
   - 持仓分析（总成本、保本价、浮动盈亏等）也是一行内联显示。

若以上都符合，说明紧凑排版已生效。

---

## 四、后端分析接口（可选）

若要验证「开始分析」能跑通：

1. 确保 `run_web.py` 已启动（终端里有「后端 API: http://localhost:8123」或 8124）。
2. 打开 **http://localhost:8888/analysis.html**，填好代码（如 `03690`）、市场（港股），可勾选「使用模拟数据」加快测试。
3. 点「开始分析」→ 应出现「分析进行中…」，稍后（或从历史报告）能看到报告。

若出现 404 或超时，请确认后端已启动且端口为 8123（或终端里提示的端口），分析页若用了 `?api=8124` 等会连到对应端口。
